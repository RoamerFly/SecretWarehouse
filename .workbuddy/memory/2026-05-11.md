## 修复快速添加浮窗拖动问题

### 问题描述
用户反馈快速添加浮窗无法拖动。

### 原因分析
快速添加浮窗混合使用了两种方法：
1. `data-tauri-drag-region` 属性
2. `getCurrentWindow().startDragging()` 方法

这两种方法冲突导致拖动失效。

### 解决方案
完全照搬快速搜索浮窗 (`QuickSearchWindow.tsx`) 的拖动实现：

1. **移除** `getCurrentWindow` 导入
2. **删除** `handleTitleMouseDown` 函数
3. **移除** 标题栏 div 的 `onMouseDown={handleTitleMouseDown}`
4. **保留** `data-tauri-drag-region` 属性
5. **保留** 子元素的 `pointer-events-none` 类名
6. **保留** 按钮的 `onMouseDown={(e) => e.stopPropagation()}`

### 修改的文件
- `src/components/QuickAddWindow.tsx`：移除 startDragging 相关代码，统一使用 data-tauri-drag-region

### Git 提交
- 提交消息：`修复快速添加浮窗拖动实现，完全照搬快速搜索浮窗的方式`
- 已推送到 GitCode (`origin main`)

## 再次修复快速添加浮窗拖动问题（使用 startDragging）

### 问题描述
用户反馈"没有解决，继续修复拖拽问题，照搬快速搜索都不会吗"。

### 原因分析
`data-tauri-drag-region` 属性在 QuickAddWindow 中没有生效（尽管 QuickSearchWindow 中使用正常）。

### 解决方案
改用 `startDragging()` 方案（与快速搜索的 `data-tauri-drag-region` 方案不同，但更可靠）：

1. **添加** `getCurrentWindow` 导入
2. **添加** `windowRef = useRef(getCurrentWindow())`
3. **添加** `handleTitleMouseDown` 函数调用 `windowRef.current.startDragging()`
4. **添加** `onMouseDown={handleTitleMouseDown}` 到标题栏 div
5. **保留** 按钮的 `onMouseDown={(e) => e.stopPropagation()}`
6. **移除** `data-tauri-drag-region` 属性
7. **移除** `pointer-events-none` 类名

### 修改的文件
- `src/components/QuickAddWindow.tsx`：完全重写，使用 startDragging() 实现拖动
